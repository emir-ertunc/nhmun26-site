# Deployment Runbook

This project deploys to Cloudflare Pages with Pages Functions and D1.

## Build Settings

- Framework preset: Vite
- Build command: `npm run check`
- Build output directory: `dist`
- Production branch: `main`

`npm run check` runs formatting, linting, the Vite production build, and a
separate TypeScript check for Cloudflare Pages Functions.

## Cloudflare Resources

Create one D1 database for production:

```bash
npx wrangler d1 create nhmun26-applications
```

Copy the returned `database_id` into `wrangler.toml`, replacing
`replace-after-d1-create`.

Apply migrations locally when testing Pages Functions:

```bash
npx wrangler d1 migrations apply nhmun26-applications --local
```

Apply migrations to production before opening applications publicly:

```bash
npx wrangler d1 migrations apply nhmun26-applications --remote
```

## Environment Variables

Set these in the Cloudflare Pages project settings:

```text
APP_ENV=production
PUBLIC_SITE_URL=https://your-domain.example
VITE_TURNSTILE_SITE_KEY=<turnstile-site-key>
TURNSTILE_SECRET_KEY=<turnstile-secret-key>
```

Keep `TURNSTILE_SECRET_KEY` private. `VITE_TURNSTILE_SITE_KEY` is intentionally
public because it is bundled into the browser app.

## Domain And DNS

Buy the domain in Cloudflare Registrar or add an existing domain to Cloudflare.
Then attach it under the Pages project's custom domains screen. Cloudflare will
manage the required DNS records once the custom domain is added.

## Security Headers

Production static responses use `public/_headers` for:

- content type sniffing protection
- referrer policy
- frame embedding restrictions
- a Content Security Policy that allows Cloudflare Turnstile
- long-term immutable caching for hashed Vite assets

Cloudflare Pages does not apply `_headers` rules to responses generated by
Pages Functions, so API responses set their own no-store JSON headers in code.

## Release Checklist

1. `npm run check`
2. Confirm `wrangler.toml` has the real D1 `database_id`
3. Apply D1 migrations with `npx wrangler d1 migrations apply nhmun26-applications --remote`
4. Set Cloudflare Pages environment variables
5. Deploy `main`
6. Open the production site and submit one test application
7. Check D1 for the inserted application row
